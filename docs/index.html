Introducción

¿Por qué Cloud Computing?

Los servidores locales cuentan con varias desventajas para los desarrolladores como por ejemplo lo caro que son estos ya que requieren de un constante mantenimiento y una capacidad de almacenamiento fijo, el cual se debe pagar con anticipación. Este tipo de servidores también necesitan mucho espacio físico, tienen pocos chances de recuperar información perdida en errores de desarrollo, están limitados a la colaboración de pocas personas y poseen otros problemas que pueden ser incómodos principalmente en empresas grandes que deben guardar mucha información en los servidores.

Para solucionar estos inconvenientes la propuesta perfecta fue el Cloud Computing, este forma de servidor evita todos los problemas mencionados del servidor tradicional. Esta tecnología lleva ya varios años en desarrollo y varias compañías han funcionado como proveedores de este servicio. Muchos de los productos ofrecidos por estas compañías se diferencian por algunos detalles los cuales serán explicados en esta investigación.

Descripción del problema

Cloud computing, o computación en la nube, es una tecnología que permite acceso remoto a softwares, almacenamiento de archivos y procesamiento de datos por medio de Internet, siendo así, una alternativa a la ejecución en una computadora personal o servidor local. La siguiente investigación tiene como propósito brindar un acercamiento al concepto de Cloud Computing así como comparar diferentes proveedores de servicio, específicamente PaaS.

Para ello, se hará una recopilación de datos de carácter bibliográficos de tal forma que se abarque la definición e historia de Cloud Computing, conceptos relevantes relacionadas a este servicio así como la explicación de modelos de despliegue y modelos de servicio que este abarca.

Además, se hará una comparativa de tres principales proveedores de Cloud Computing con respecto a su oferta general, posicionamiento en el mercado y beneficios para los developers. Por último, se desarrollará una aplicación sencilla en el PaaS de tres proveedores distintos, para así poder tener una comprensión más cercana de los términos y brindar un análisis experimental que compare los tres servicios utilizados.
